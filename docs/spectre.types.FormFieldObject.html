FormFieldObject

  • typedef
spectre.types.FormFieldObject

{Object}

 

This is an object used by the form-widget to control tye type of field that is displayed and how the field is displayed. This can either be a string representing the field name or an object with the properties described below.

The value for properties depends on the type of field used. For example, the file-field requires a url inside the property.

Object

Properties

  1. name {String}

    The name of the field property

  2. alias {String}

    A label to display for the field

  3. placeholder {String}

    The field placeholder to display when no text is entered

  4. type {String}

    The type of field template to use.

  5. template {can.view.renderer}

    A template partial which gets passed the formFieldObject. You can create a renderer using can.stache(template), or importing the template with steal. If this is provided, the type will be ignored.

  6. properties {Object}

    Additional properties to pass to the field template. For example, options is a property existing in the select template

  7. value {String}

    The value stored in the formObject. This is provided by the form-widget internally
